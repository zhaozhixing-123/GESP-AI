# GESP.AI 产品需求文档（PRD）

## 1. 项目概述

### 1.1 产品名称
GESP.AI — GESP智能刷题平台

### 1.2 产品定位
面向GESP（全国青少年编程等级考试C++方向）考生的智能刷题Web平台，提供真题练习、在线判题和AI辅导功能。

### 1.3 目标用户
- 初期：开发者自用
- 后期：参加GESP考试的青少年学生

### 1.4 核心价值
- 集中管理GESP 1-8级编程真题
- 在线编写、提交、判题
- AI老师引导式辅导，培养独立思考能力（不直接给答案）

---

## 2. 功能需求

### 2.1 用户系统

| 功能 | 描述 |
|------|------|
| 注册 | 用户名 + 密码注册 |
| 登录 | 用户名 + 密码登录 |
| 登出 | 退出当前账号 |
| 角色 | 分为 admin（管理员）和 user（普通用户）两种角色 |

> 初期保持简单，不需要邮箱验证、第三方登录等。
> 第一个注册的用户自动成为管理员，后续注册为普通用户。

### 2.2 题库系统

| 功能 | 描述 |
|------|------|
| 题目列表 | 按GESP级别（1-8级）分类浏览 |
| 题目筛选 | 按级别筛选题目 |
| 题目详情 | 显示题目描述、输入输出格式、样例数据 |
| 题目状态 | 标记已通过/未通过/未尝试 |

**数据来源：** 从洛谷平台爬取GESP编程真题，包括：
- 题目标题
- 题目描述（含公式、图片）
- 输入输出格式说明
- 样例输入输出
- 测试数据（用于判题）

### 2.3 代码编辑与判题

| 功能 | 描述 |
|------|------|
| 代码编辑器 | 在线C++代码编辑器，支持语法高亮 |
| 提交判题 | 提交代码到Judge0进行编译运行 |
| 判题结果 | 显示AC/WA/TLE/CE等状态及详细信息 |
| 提交记录 | 查看历史提交记录 |

**判题服务：** 使用Judge0开源在线判题服务
- 支持C++语言
- 对比用户输出与标准答案
- 返回运行时间、内存占用

### 2.4 AI老师

| 功能 | 描述 |
|------|------|
| 对话入口 | 每道题旁边有"问AI老师"按钮 |
| 思路提示 | 给出解题方向，如"可以考虑用递归" |
| 分步引导 | 像真人老师一样逐步提问，引导思考 |
| 指出错误 | 分析用户代码，指出问题但不给正确写法 |
| 不给答案 | 系统提示词严格限制，不输出完整解题代码 |

**AI服务：** 调用Claude API
- System Prompt中严格限制不给完整答案
- 上下文包含当前题目信息和用户代码
- 支持多轮对话

### 2.5 提示词管理（管理员功能）

| 功能 | 描述 |
|------|------|
| 提示词列表 | 查看系统中所有AI提示词，包括名称、用途、内容 |
| 新增提示词 | 创建新的提示词模板 |
| 编辑提示词 | 在线修改提示词内容，实时生效 |
| 删除提示词 | 删除不再使用的提示词 |
| 提示词分类 | 按用途分类，如：系统提示词、思路引导、错误分析、分步引导等 |

> 仅系统管理员（role=admin）可见此功能。普通用户无法访问。

**提示词类型：**
- **AI老师 System Prompt** — 控制AI老师的整体行为和规则
- **思路提示模板** — 当用户请求思路提示时的引导模板
- **错误分析模板** — 当用户提交代码有误时的分析引导模板
- **分步引导模板** — 逐步引导学生的对话模板

**设计要点：**
- 提示词存储在数据库中，管理员通过前端页面修改，无需改代码即可调优AI行为
- 每个提示词支持变量占位符（如 `{{problem_title}}`、`{{user_code}}`），运行时自动替换
- 修改后立即生效，无需重启

### 2.6 错题本

| 功能 | 描述 |
|------|------|
| 自动收录 | 提交判题结果为WA的题目自动加入错题本 |
| 查看错题 | 列表查看所有错题，点击进入题目 |
| 移除错题 | 通过该题后自动从错题本移除 |

## 3. 页面结构

```
/ (首页/登录)
/register (注册页)
/problems (题目列表)
/problems?level=3 (按级别筛选)
/problems/:id (题目详情 + 代码编辑器 + AI老师)
/wrongbook (错题本)
/admin/prompts (提示词管理 - 仅管理员)
```


## 4. 技术方案

### 4.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Next.js (React) |
| UI组件 | Tailwind CSS |
| 代码编辑器 | Monaco Editor (VS Code同款) |
| 后端 | Next.js API Routes |
| 数据库 | SQLite (via Prisma ORM) |
| AI服务 | Claude API |
| 判题服务 | Judge0 (自托管或云服务) |
| 部署 | Vercel (前端) + 自有服务器 (Judge0) |

### 4.2 数据库设计

**User 表**
- id, username, password_hash, role (admin/user), created_at

**Problem 表**
- id, luogu_id, title, level (1-8), description, input_format, output_format, samples, test_cases

**Submission 表**
- id, user_id, problem_id, code, language, status, time_used, memory_used, created_at

**WrongBook 表**
- id, user_id, problem_id, added_at

**ChatHistory 表**
- id, user_id, problem_id, role (user/assistant), content, created_at

**Prompt 表**
- id, name, category (system/hint/error_analysis/step_guide), content, variables, updated_at

### 4.3 爬虫方案

- 使用Node.js脚本爬取洛谷GESP题目
- 爬取内容：题目描述、输入输出格式、样例、测试数据
- 一次性爬取后存入数据库，后续手动更新

### 4.4 AI老师 System Prompt 设计要点

```
你是GESP.AI的AI编程老师，帮助学生学习C++和GESP考试。

核心规则：
1. 绝对不能给出完整的解题代码
2. 绝对不能直接说出最终答案
3. 用引导式提问帮助学生自己想出解法
4. 可以解释概念、给思路方向、指出代码错误
5. 语言简洁，适合小学到初中学生理解
```

## 5. 开发计划

### 第一阶段：基础框架
- 项目初始化 (Next.js + Tailwind + Prisma)
- 用户注册登录
- 数据库设计与创建

### 第二阶段：题库
- 洛谷爬虫脚本
- 题目列表页（按级别筛选）
- 题目详情页

### 第三阶段：代码编辑与判题
- 集成Monaco Editor
- 对接Judge0判题
- 提交记录

### 第四阶段：AI老师
- 集成Claude API
- 对话界面
- System Prompt调优

### 第五阶段：错题本
- 自动收录逻辑
- 错题本页面
- AC后自动移除

---

## 6. 非功能需求

- **安全性：** 密码bcrypt加密存储，防SQL注入
- **性能：** 单用户使用，性能要求不高
- **可扩展性：** 后续可增加选择题、更多语言支持
